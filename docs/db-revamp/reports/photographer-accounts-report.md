# PR-08: Photographer portal accounts (infrastructure only)

## Purpose

Provide **separate** credentials for a future photographer portal. These accounts are **not** Better Auth users, **not** subscribers, and **not** admin users.

## Table

`photographer_accounts` (see `apps/api/src/db/schema/photographer-accounts.ts`):

- One row per photographer (unique `photographer_id`).
- `username`: deterministic `ph_` + zero-padded numeric `legacy_photographer_id` (no email as username).
- `password_hash`: **scrypt** string (`$scrypt$n=16384,r=8,p=1$...`), never plaintext.
- `must_change_password`: default `true` for generated credentials.
- `status`: `ACTIVE` | `DISABLED` | `LOCKED`.

Helper: `apps/api/src/lib/auth/photographer-password.ts` uses `@noble/hashes/scrypt.js`, so the same `$scrypt$` hashes can be generated by CLI scripts and verified by Cloudflare Worker runtime code. The legacy script helper at `apps/api/scripts/lib/photographer-password-hash.ts` re-exports this runtime-compatible helper.

## Generation

```bash
pnpm --dir apps/api photographers:generate-accounts -- --mode=current-imported
pnpm --dir apps/api photographers:generate-accounts -- --mode=all-active
pnpm --dir apps/api photographers:generate-accounts -- --mode=all
```

- **`--mode` is required** (no silent default).
- **`current-imported`**: photographers with at least one `image_assets` row (and not `DELETED` unless `--include-deleted`).
- **`all-active`**: `photographers.status = 'ACTIVE'`.
- **`all`**: all photographers except `DELETED` unless `--include-deleted`.
- **`--reset-passwords`**: overwrite hash for existing accounts and emit new CSV rows.
- **`--dry-run`**: no DB writes; no CSV with secrets; prints planned counts only.
- **`--output <path>`**: optional CSV path (default under `apps/api/exports/photographer-accounts/`).

CSV columns include `temporary_password` and `login_scope` = `PHOTOGRAPHER_PORTAL_ONLY`. **Store the CSV securely; treat as single-use distribution.**

Legacy `ppassword` and `photographer_profiles` duplicates are **not** used.

## Validation

```bash
pnpm --dir apps/api db:validate:photographer-accounts
```

## Deferred

- Photographer portal **login UI**, session, and API routes are **out of scope** for this PR.

## Risks

- No portal auth yet; credentials are meaningless until a login boundary exists.
- Rerunning generation **without** `--reset-passwords` skips existing accounts (no password in CSV).
- CSV contains plaintext passwords once; do not commit or log it.
